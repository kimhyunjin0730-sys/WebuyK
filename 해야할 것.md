## ☁️ Google Cloud Run 배포 (API 서버)

API 서버는 Cloud Run, 웹은 Vercel 그대로. 월 1만 요청 미만이면 **무료** (free tier: 월 200만 요청, 360K GiB-초, 180K vCPU-초).

### 1. GCP 계정 준비

- [ ] https://console.cloud.google.com 접속 (Google 계정 필요)
- [ ] 처음이면 **신용카드 등록** 필요 — 무료 한도 초과 시 청구, 한도 내 $0
- [ ] 신규 프로젝트 생성 (예: `wbk-api-prod`) → 프로젝트 ID 메모

### 2. gcloud CLI 설치 & 로그인

- [ ] https://cloud.google.com/sdk/docs/install 에서 설치
- [ ] 로그인 + 프로젝트 설정:
  ```bash
  gcloud auth login
  gcloud config set project wbk-api-prod  # 본인 프로젝트 ID로 교체
  ```
- [ ] 필요 API 활성화:
  ```bash
  gcloud services enable run.googleapis.com cloudbuild.googleapis.com artifactregistry.googleapis.com
  ```

### 3. 환경변수를 Secret Manager에 저장 (권장)

- [ ] 민감한 값들을 Secret으로 등록:
  ```bash
  echo -n "실제_값" | gcloud secrets create DATABASE_URL --data-file=-
  echo -n "실제_값" | gcloud secrets create JWT_SECRET --data-file=-
  echo -n "새로_발급한_키" | gcloud secrets create ANTHROPIC_API_KEY --data-file=-
  echo -n "https://your-vercel-app.vercel.app" | gcloud secrets create WEB_ORIGIN --data-file=-
  ```
  (OAuth 쓰면 `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET`, `KAKAO_CLIENT_ID`, `KAKAO_CLIENT_SECRET`도 동일 방식)

### 4. 배포

- [ ] 프로젝트 루트에서:
  ```bash
  gcloud run deploy wbk-api \
    --source . \
    --dockerfile apps/api/Dockerfile \
    --region asia-northeast3 \
    --platform managed \
    --allow-unauthenticated \
    --memory 2Gi \
    --cpu 1 \
    --min-instances 0 \
    --max-instances 3 \
    --timeout 60 \
    --update-secrets DATABASE_URL=DATABASE_URL:latest,JWT_SECRET=JWT_SECRET:latest,ANTHROPIC_API_KEY=ANTHROPIC_API_KEY:latest,WEB_ORIGIN=WEB_ORIGIN:latest
  ```
  - `--min-instances 0`: 미사용 시 인스턴스 0개 → 요금 $0
  - `--max-instances 3`: **비용 폭탄 방지 필수**
  - `--memory 2Gi`: Playwright Chromium 돌리려면 최소 2GB
  - `asia-northeast3`: 서울 리전 (한국 사이트 스크래핑 속도↑)
- [ ] 배포 완료되면 출력되는 Service URL 메모 (예: `https://wbk-api-xxxxx-du.a.run.app`)

### 5. 비용 폭탄 방지 장치 🛡️

- [ ] **Billing Budget Alert 설정**
  - Console → Billing → Budgets & alerts → Create budget
  - Amount: $1 또는 $5 (원하는 값)
  - Alert: 50% / 90% / 100% 이메일 알림
- [ ] **Max instances 확인** — 위 deploy 명령에 `--max-instances 3` 포함됨 ✓
- [ ] 공개 엔드포인트 보호 고민: `--allow-unauthenticated`로 뒀기 때문에 누구나 호출 가능 → 악성 트래픽 대비 나중에 API 키 미들웨어 또는 Cloud Armor 추가 고려
