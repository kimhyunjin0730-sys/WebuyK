# 해야할 것

상품 인식 3-모드 (빠른 URL / 정밀 Playwright / 스크린샷 Vision) 구현 후 사용자가 직접 마무리해야 하는 항목.

## 🔴 긴급 — 보안

- [ ] **노출된 Anthropic API 키 폐기**
  - 채팅에 평문으로 두 번 노출된 키 (`sk-ant-api03-55wDZ...AAA`)는 사용 불가로 간주하고 즉시 삭제
  - https://console.anthropic.com/settings/keys → 해당 키 옆 `...` → **Delete**
  - 새 키 발급 ("Create Key")
  - 새 키는 **절대 채팅에 붙여넣지 말고** `.env` 또는 Render/Vercel 대시보드에만 입력

## 로컬 환경 셋업

- [ ] 의존성 설치
  ```bash
  pnpm install
  ```
- [ ] Playwright Chromium 설치
  ```bash
  pnpm --filter @wbk/api exec playwright install chromium
  ```
- [ ] `apps/api/.env`에 새 API 키 추가
  ```env
  ANTHROPIC_API_KEY=새로_발급한_키
  ```
- [ ] 로컬에서 동작 확인
  ```bash
  pnpm dev
  ```
  - 빠른 URL 모드: 네이버 스마트스토어 URL로 테스트
  - 정밀 URL 모드: 쿠팡 URL로 테스트 (5~15초 소요)
  - 스크린샷 모드: 아무 상품 페이지 캡처해서 업로드

## Render 배포 (API 서버)

- [ ] https://dashboard.render.com 접속 → **New → Blueprint**
- [ ] 이 GitHub 레포 연결 (`render.yaml` 자동 인식)
- [ ] 환경변수 입력 (`sync: false` 항목 전부):
  - [ ] `DATABASE_URL` — MySQL 커넥션 스트링
  - [ ] `JWT_SECRET` — 강한 랜덤 문자열 (예: `openssl rand -hex 32`)
  - [ ] `WEB_ORIGIN` — `https://your-vercel-app.vercel.app`
  - [ ] `ANTHROPIC_API_KEY` — 새로 발급한 키
  - [ ] (선택) `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET`
  - [ ] (선택) `KAKAO_CLIENT_ID`, `KAKAO_CLIENT_SECRET`
- [ ] Deploy 클릭 → 빌드 완료까지 대기 (~10분, 첫 배포)
- [ ] 발급된 Render URL 메모 (예: `https://wbk-api.onrender.com`)

## Vercel (Web) 환경변수 업데이트

- [ ] Vercel 프로젝트 → Settings → Environment Variables
- [ ] `NEXT_PUBLIC_API_URL` 추가/수정 → Render URL + `/api`
  - 예: `https://wbk-api.onrender.com/api`
- [ ] Vercel 재배포 (Deployments → 최신 → Redeploy)

## 마무리 정리 (Render 배포 검증 후)
